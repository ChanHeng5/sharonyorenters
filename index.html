Page Layout

This section explains about types of page layout styles that has been identified in Sharon.

  • All types of page layout will use the same header as shown in the image below.
  • Below are the types of page layouts that have been identified in Sharon.

Type 1 - Homepage

  • Homepage is the first page user will see when visiting our website and after they login.
  • This layout consist of few elements as below:
    • Tagline with navigation buttons for owner and renter section.
    • Banner section to display any advertisments and promotions.
    • Categories section for user to browse item by category.
    • Hot items section
  • The example of the layout style is as in the image below.

Type 2 - General pages.

  • General pages are the pages that show information about the company and business operation.
  • Example of general pages are About Us, How It Works, Privacy Policy, Terms of Use, FAQ anf Contact Us.
  • This layout consist of few elements as below:
    • Breadcrumb under the header bar. Can refer to breadcrumb section for more details
    • Menu section on the left side for user to navigate to any specific page.
    • Content section on the right with page title(can refer Headings section for more details) at the top.
  • The example of the layout style is as in the image below.

Type 3 - My Account pages.

  • My Account pages are the pages for user to manage their own information and details.
  • Example of My Account pages are Profile, Change Password, Address Book, Security Deposit Wallet, My Orders and Reviews page.
  • This layout consist of few elements as below:
    • Breadcrumb under the header bar. Can refer to breadcrumb section for more details
    • Menu section on the right side for user to navigate to any specific page with menu title and current user type at the top.
    • Content section on the left with page title(can refer Headings section for more details) at the top.
  • The example of the layout style is as in the image below.

Type 4 - Preview Item page.

  • Preview Item page is a page for user to view details of an item. It has its own layout style which is diiferent from other page.
  • This layout consist of few elements as below:
    • Breadcrumb under the header bar. Can refer to breadcrumb section for more details
    • Details of the item which is item image on the left and item description on the right. The heading type for the item name is different from other "page title" style.(Can refer at heading section).
    • Description, Specification and Reviews of the item are located at the bottom of the page.
  • The example of the layout style is as in the image below.

Type 5 - Common Pages.

  • Common pages are pages that doesn't have any specifix layout where it only has breadcrumb, page-title/section-title and its content.
  • Examples of page which are using this layout are Item listing, Promotions, Rating & Review and Checkout
  • This layout consist of few elements as below:
    • Breadcrumb under the header bar. Can refer to breadcrumb section for more details
    • Page title.(Can refer headings section for more details).
    • Content section.
  • The example of the layout style is as in the image below.

Headings

This section explains about heading tag information and the usage of the heading

Heading font size and weight

All developers must follow the heading tag's font size and weight that have been defined in styles.css as below. Please do not overwrite it with different font size and weight.

<h1> tag
  • Font Size: 32px
  • Font Weight: 600
<h2> tag
  • Font Size: 28px
  • Font Weight: 600
<h3> tag
  • Font Size: 24px
  • Font Weight: 600
<h4> tag
  • Font Size: 18px
  • Font Weight: 600
<h5> tag
  • Font Size: 14px
  • Font Weight: 600
<h6> tag
  • Font Size: 12px
  • Font Weight: 600

The usage of the heading

1. Page Title / Section Title
  • Page title / Section title need to use <h4> heading and must be within <div> element with class .page-title.
  • This title style should be applied at all pages except in Preview Item page.
Code Example
<div class="page-title">
<h4>Electronic Devices</h4>
</div>
Output Example

Electronic Devices

2. Item title in Preview Item Page
  • Item title in Preview item page need to use <h1> heading tag.
3. Paragraph title in article
  • Title for specific paragraph (if any) has to use <h4> heading tag.
Code Example
<h4> Serving your Desires, The Sustainable Way </h4>
<p> YORenters.com supports the idea of renting instead of buying and contributes to building an eco-friendly and sustainable environment by reducing wastage ... </p>
Output Example

Serving your Desires, The Sustainable Way

YORenters.com supports the idea of renting instead of buying and contributes to building an eco-friendly and sustainable environment by reducing wastage...

Labels

This section explains about labels information and the usage of the labels.

  • Label usually to be used in form or to display information.
  • Use <label></label> html tag for label.
  • Text for label must end with <space> followed by colon ":" (eg. ).
    • Code Example
    <label>Gender :</label>
    Output Example
  • For form label use Astersik "*" with red color by using <span class="text-danger ml-1">*</span> before colon to indicate that the field is cumpolsory (eg. )
    • Code Example
    <label>Email<span class="text-danger ml-1">*</span> :</label>
    Output Example
  • Defaul font size for label is 14px and font weight is 600. Please avoid from changing this size.

Buttons

This section explains about button information and the usage of the button.

  • Only first letter of the button word is capital.(eg. Register, Login, Edit Profile).
  • All action buttons should be at the right side of the page.
  • Icon (if any) should be on the left of the text except for the “next” button should be on the right of the text.

Button Colors

1. Login / Register / Add/ Create / New / Next/ Proceed / Submit / Save / Update / Okay
  • All of these buttons or any similar meaning button need to use the same class which is btn btn-hero.
Code Example
<button type="button" class="btn btn-hero"><i class="fa fa-save mr-1"></i>Submit</button>
Output Example
2. Edit
  • Use btn btn-edit class for edit button.
Code Example
<button type="button" class="btn btn-edit"><i class="fa fa-save mr-1"></i>Submit</button>
Output Example
3. Delete / Remove
  • Use btn btn-danger class for Delete, Remove or any similar meaning buttons.
Code Example
<button type="button" class="btn btn-danger"><i class="fa fa-trash-o mr-1"></i>Submit</button>
Output Example
4. Back / Cancel
  • Use btn btn-back class for Back, Cancel or any similar meaning buttons.
Code Example
<button type="button" class="btn btn-back"><i class="fa fa-arrow-circle-left mr-1"></i>Back</button>
Output Example
5. Search / Select / Find / Preview / Use / any helper function
  • Use btn btn-info class for Search, Select, Find, Preview or any similar meaning buttons.
Code Example
<button type="button" class="btn btn-info"><i class="fa fa-search mr-1"></i>Preview</button>
Output Example

Icons for button

1. Login
  • Use <i class="icon-lock icons mr-1"></i> icon code for Login button.
Code Example
<button type="button" class="btn btn-hero"><i class="icon-lock icons mr-1"></i>Login</button>

Anchor

This section explains about anchor information and the usage of the anchor.

  • Use <a></a> html tag for anchor
  • Default color for anchor tag is black (Color code: #333333 and hover color is yorenters main color var(--main-color). Please avoid from changing these colors.
    • Code Example
    <a>Click here</a>
    Output Example
    Click here
  • Use href attribute href="#" to make cursor turn to pointer format and to navigate to other page or section.
    • Code Example
    <a href="#">Click here</a>
    Output Example
    Click here

Anchor in article

  • For article, color for the anchor will use yorenters main color var(--main-color) and for hover use yorenters second color var(--second-color).
  • To implement this, use .anchor-article class in <a> tag
    • Code Example
    <a class="anchor-article" href="#">Click here</a>
    Output Example
    Click here

Form Validation Message

  • Form validation message in a form should be in a red colour by using <span></span> and text-danger class and located under the respective textbox.
  • All validation message need to folow format as below.
    • Control Type Message Format Sample Message
    Textbox Enter "Display Name" Enter first name.
    Enter last name.
    Enter email address.
    Enter item name.
    Textbox amount
    * Validation on range     		Amount must be more than RM "amount" Amount must be more than RM 10.00
    Amount must be less than RM 1000.00
    Textbox number
    * Validation on range     		Permitted "Display Name" is in between "min value" and "max value" Permitted rental days is in between 1 days and 30 days only.
    Dropdown
    		 Choose "Option Name" Choose item category.
    Choose payment mode.
    Calendar
    		 Select "Display Name" Select date of birth.
    Select item expiry date.
    File upload
    		 That file type isn’t supported for
    "Display Name".
    Supported types are: "Supported type"    		 That file type isn’t supported for item image. Supported types are: jpg, jpeg, png.

Form Validation Summary Message

  • Validation summary is to summarize error in a form which located at the top of the form when it is shown.
  • Validation summary message need to be wrap with bootstrap alert box as below:
    Code Example
    <div class="alert alert-danger">
    Validation summary here.
    </div>
    Output Example
    Validation summary here.
